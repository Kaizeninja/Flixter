Lesson 33: Styling the Instructor Course Overview Page

In the previous lessons, we added support for the features users need on the instructor course overview page. We focused our efforts on building the functionality and making our code work. In this lesson we will put on our web designer hats and make this page look awesome.

Before we dive in and change the styles, it makes sense to take a step back and revisit the wireframes. That way we can make sure we're building what we set out to in the first place. Take a look at the wireframes here:

Teacher Course Overview Page


Making the Sortable List Items Display a Font Awesome Icon Rather Than a Bullet Point

Challenge!

Our first step is adding the Font Awesome to our project like we did in Lesson 15 of Nomster. Try to add Font Awesome on your own by looking for the gem on Google and walking through the instructions on Github. If you can't figure it out then follow the instructions in Nomster.

One inconsistency between our web application and what appears on the wireframes is that the sections' and lessons' list items have a bullet point next to them on our application, but not on the wireframes. In order to remove the bullet points we will need to add a line of CSS. If you do a Google search with the query Remove li Bullets CSS one of the first search results will show us the CSS we need to add. Edit app/assets/stylesheets/master.scss and add the following line: 

ul {
  list-style-type: none;
}

Save the file. Navigate to the instructor course overview page and refresh the page. The bullet points should disappear.

The wireframes include triple vertical dots to indicate items are sortable. Let's add a similar icon to our application to make this indication.

Return to the Font Awesome Icon List and browse through the collection. Take a moment and find the icon labeled ellipsis-v. This icon will indicate to our user (instructor) that the list is sortable, so add the icon whenever we have a sortable list. Edit app/views/instructor/courses/show.html.erb to include the classes fa and fa-ellipsis-v whenever we have a sortable list item: 

<ul class="sections">
<% current_course.sections.rank(:row_order).each do |section| %>
  <li data-update-url="<%= instructor_section_path section %>">
    <span class="fa fa-ellipsis-v"></span>
    <%= section.title %>
    <%= link_to 'New Lesson', new_instructor_section_lesson_path(section), class: 'float-right btn btn-primary btn-sm' %>
    <ul class="lessons">
      <% section.lessons.rank(:row_order).each do |lesson| %>
        <li data-update-url="<%= instructor_lesson_path lesson %>">
          <span class="fa fa-ellipsis-v"></span>
          <b><%= lesson.title %></b>
          -
          <%= lesson.subtitle %>
        </li>
      <% end %>
    </ul>
    <br class="clr" />
  </li>
<% end %>
</ul>

Save the file and refresh the page. Notice how the icons appear next to the sortable items.

Adjusting the Styles for the Sortable Sections

On the wireframes each section has a border and this makes each section seem to be enclosed in a box. Let's lift the CSS we used on our booyah-box for the border and add it to a new CSS selector section-li, which we will apply to the list item's HTML tag in a moment.

Edit app/stylesheets/master.scss to look like this:

.section-li {
  border: 1px solid;
  border-color: #e4e4e4 #bebebd #bebebd #e4e4e4;
}

Save the file.

Now apply the section-li class to the HTML list item element in app/views/instructor/courses/show.html.erb: 

<ul class="sections">
<% current_course.sections.rank(:row_order).each do |section| %>
  <li data-update-url="<%= instructor_section_path section %>" class="section-li">
    <span class="fa fa-ellipsis-v"></span>
    <%= section.title %>
    <%= link_to 'New Lesson', new_instructor_section_lesson_path(section), class: 'float-right btn btn-primary btn-sm' %>
    <ul class="lessons">
      <% section.lessons.rank(:row_order).each do |lesson| %>
        <li data-update-url="<%= instructor_lesson_path lesson %>">
          <span class="fa fa-ellipsis-v"></span>
          <b><%= lesson.title %></b>
          -
          <%= lesson.subtitle %>
        </li>
      <% end %>
    </ul>
    <br class="clr" />
  </li>
<% end %>
</ul>

Save the file and refresh the page. Notice how a border appears around each of the sections.

Wrap the title for the section inside a div with the class of header. We will adjust the font size of this in a moment.

Edit app/views/instructor/courses/show.html.erb to add the CSS class into the HTML:

<ul class="sections">
<% current_course.sections.rank(:row_order).each do |section| %>
  <li data-update-url="<%= instructor_section_path section %>" class="section-li">
    <div class="header">
      <span class="fa fa-ellipsis-v"></span>
      <%= section.title %>
    </div>
    <%= link_to 'New Lesson', new_instructor_section_lesson_path(section), class: 'float-right btn btn-primary btn-sm' %>
    <ul class="lessons">
      <% section.lessons.rank(:row_order).each do |lesson| %>
        <li data-update-url="<%= instructor_lesson_path lesson %>">
          <span class="fa fa-ellipsis-v"></span>
          <b><%= lesson.title %></b>
          -
          <%= lesson.subtitle %>
        </li>
      <% end %>
    </ul>
    <br class="clr" />
  </li>
<% end %>
</ul>

Save the file.

Write the CSS code that will match on the header class within an unordered list and set the font-size to 25 pixels. Edit app/assets/stylesheets/master.scss to include this line: 

ul .header {
  font-size: 25px;
}

Save the file and refresh the page. Notice how the font-size of text within the header div increased.

Unfortunately, this also increased the size of the Font Awesome icon that we're using to indicate the item is sortable. It looks too big at the size that it currently is. Add the following code to app/assets/stylesheets/master.scss to specify this particular Font Awesome icon should be only 20 pixels tall: 

ul .header .fa {
  font-size: 20px;
}

Save the file and refresh the page. That size looks better!

Right now the content within the section-li looks too close to the border we added. Edit app/assets/stylesheets/master.scss and include a padding of 20 pixels, so the content has more space: 

.section-li {
  border: 1px solid;
  border-color: #e4e4e4 #bebebd #bebebd #e4e4e4;
  padding: 20px;
}

Save the file and refresh the page. The sections on the instructor course overview page look awesome!

Adjusting the Styles for the Sortable Lessons

The styling of the sections in our web application looks very close to what we set out to build. Next let's check our wireframes and see what changes we should make to the lessons that are nested within the sections box.

Add CSS to the sortable lesson list items and make them have a black background. To do that, add the following CSS to app/assets/stylesheets/master.scss. 

.lessons li {
  background: black;
}

Save the file and refresh the page. Notice how the background color of the lesson blocks changed to black.

It's difficult to read the text of the content within the lesson box now that its background color is black. Edit app/assets/stylesheets/master.scss to change the content within these black boxes to be white: 

.lessons li {
  background: black;
  color: white;
}

Save the file and refresh the page. Notice the text within the lesson box changes color.

Each of these lesson boxes looks too close to each other and also too close to the edge of its section's border. Edit app/assets/stylesheets/master.scss to add a 10 pixel margin to the edges of the lesson boxes: 

.lessons li {
  background: black;
  color: white;
  margin: 10px;
}

Save the file and refresh the page. Notice how the change spaced out the lesson items.

Take a moment to improve the fonts that are used within the black lesson box. Edit app/assets/stylesheets/master.scss to set the font to be a thin version of Helvetica Neue with a font weight of 100: 

.lessons li {
  background: black;
  color: white;
  margin: 10px;
  font-family: "Helvetica Neue", Helvetica;
  font-weight: 100;
}

Save the file and refresh the page. Notice the impact the font change had on the overall feel of the design of the lesson box.

Add 8 pixels of padding to the lesson box, so the content within the black lesson box has more space from the edge of the box. Edit app/assets/stylesheets/master.scss to look like this: 

.lessons li {
  background: black;
  color: white;
  margin: 10px;
  font-family: "Helvetica Neue", Helvetica;
  font-weight: 100;
  padding: 8px;
}

Save the file and refresh the page. Notice how there is more space between the edge of the black lesson box and the content within it.

Move the lesson title and the lesson subtitle onto different lines. To do this edit app/views/instructor/courses/show.html.erb and wrap each in a p tag. 

<ul class="lessons">
  <% section.lessons.rank(:row_order).each do |lesson| %>
    <li data-update-url="<%= instructor_lesson_path lesson %>">
      <span class="fa fa-ellipsis-v"></span>
      <p>
        <b><%= lesson.title %></b>
      </p>
      <p>
        <%= lesson.subtitle %>
      </p>
    </li>
  <% end %>
</ul>

Save the file and refresh the page. Notice how wrapping each item in a separate p tag causes each to be on a separate line.

Position the Font Awesome icon, which indicates this is a sortable list item, to be directly to the left of the lesson title and subtitle.To do this, use the Bootstrap Grid System and place the Font Awesome icon inside a col that is 1/12th the screen width. Place the title and subtitle into a col that is 11/12ths the width. Edit app/views/instructor/courses/show.html.erb like this: 

<ul class="lessons">
  <% section.lessons.rank(:row_order).each do |lesson| %>
    <li data-update-url="<%= instructor_lesson_path lesson %>">
      <div class="row">
        <div class="col-1">
          <span class="fa fa-ellipsis-v"></span>
        </div>
        <div class="col-11">
          <p>
            <b><%= lesson.title %></b>
          </p>
          <p>
            <%= lesson.subtitle %>
          </p>
        </div>
      </div>
    </li>
  <% end %>
</ul>

Clear out the Bootstrap Grid System after specifying the col width. Add a clearing breakline to app/views/instructor/courses/show.html.erb like this: 

<ul class="lessons">
  <% section.lessons.rank(:row_order).each do |lesson| %>
    <li data-update-url="<%= instructor_lesson_path lesson %>">
      <div class="row">
        <div class="col-1">
          <span class="fa fa-ellipsis-v"></span>
        </div>
        <div class="col-11">
          <p>
            <b><%= lesson.title %></b>
          </p>
          <p>
            <%= lesson.subtitle %>
          </p>
        </div>
      </div>
      <br class="clr" />
    </li>
  <% end %>
</ul>

Save the file and refresh the page. Notice how this change pulled the Font Awesome icon to the left of the title and subtitle.

Center the title and subtitle within the div we just used by adding the text-center class to the div within app/views/instructor/courses/show.html.erb: 

<ul class="lessons">
  <% section.lessons.rank(:row_order).each do |lesson| %>
    <li data-update-url="<%= instructor_lesson_path lesson %>">
      <div class="row">
        <div class="col-1">
          <span class="fa fa-ellipsis-v"></span>
        </div>
        <div class="col-11 text-center">
          <p>
            <b><%= lesson.title %></b>
          </p>
          <p>
            <%= lesson.subtitle %>
          </p>
        </div>
      </div>
      <br class="clr" />
    </li>
  <% end %>
</ul>

Save the file and refresh the page. We're getting very close to where we want to be. Next we'll apply a couple more subtle changes to the look and feel to bring it all the way.

Adjust the font size for the content within the black lesson box. Edit app/assets/stylesheets/master.scss to set the font size within the box to be 15 pixels. 

.lessons li {
  background: black;
  color: white;
  margin: 10px;
  font-family: "Helvetica Neue", Helvetica;
  font-weight: 100;
  padding: 8px;
  font-size: 15px;
}

Save the file and refresh the page.

Make the title of the lesson more prominent than the subtitle by giving it a larger font. The title is wrapped in the b tag so we can add a CSS property for that. Edit app/assets/stylesheets/master.scss like this: 

.lessons li b {
  font-size: 25px;
}

Save the file and refresh the page.

Adjust the font weight of the title so it's very thin. Set the font weight to 100 in app/assets/stylesheets/master.scss like this: 

.lessons li b {
  font-size: 25px;
  font-weight: 100;
}

Save the file and refresh the page. It looks almost perfect, but it will look better if we tweak how we space out the elements.

Tighten the margins within the p tag by adding the following to app/assets/stylesheets/master.scss: 

.lessons li p {
  margin: 0;
}

Save the file and refresh the page.

The Font Awesome icon within the black lesson box is too high. Push it down a little by adding a br tag to app/views/instructor/courses/show.html.erb: 

<div class="col-1">
	<br />
	<span class="fa fa-ellipsis-v"></span>
</div>

Save the file and refresh the page.

Add clearing breaklines after the new button to pull the header to the left. Edit app/views/instructor/courses/show.html.erb to include these br tags: 

<div class="header float-left">
  <span class="fa fa-ellipsis-v"></span>
  <%= section.title %>
</div>
<%= link_to 'New Lesson', new_instructor_section_lesson_path(section), class: 'float-right btn btn-primary btn-sm' %>
<br class="clr" />
<br class="clr" />

Save the file and refresh the page.

Adjust the margins around the section-li to be 20 pixels by adding the following line to app/assets/stylesheets/master.scss: 

.section-li {
  border: 1px solid;
  border-color: #e4e4e4 #bebebd #bebebd #e4e4e4;
  padding: 20px;
  margin: 20px;
}
